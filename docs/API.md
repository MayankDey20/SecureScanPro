# API Documentation

## Overview

The SecureScan Pro backend is built with FastAPI. It provides a RESTful API for scheduling and managing security scans, retrieving scan results, and downloading reports.

## Authentication

Authentication is handled via Supabase JWT tokens. Include the token in the Authorization header of your requests.

`Authorization: Bearer <your_supabase_jwt>`

## Core Endpoints

### Scans

- **POST /api/v1/scans/start**
  Initiates a new security scan against a target URL.
  Payload parameters include `target_url`, `scan_type`, and specific scanning configuration flags.
  Returns a `scan_id` which can be used to track progress.

- **GET /api/v1/scans/{scan_id}**
  Retrieves the current status and summary results of a specific scan.

- **GET /api/v1/scans**
  Returns a paginated list of historical scans associated with the authenticated user.

### Vulnerabilities

- **GET /api/v1/vulnerabilities**
  Retrieves a list of vulnerabilities detected across all the user's scans. Supports filtering by severity, status, and target.

### Analytics

- **GET /api/v1/analytics/dashboard**
  Fetches aggregated statistics for the user dashboard, including total scans, average threat scores, and vulnerability trends.

## Real-time WebSockets

- **WS /api/v1/ws/scans/{scan_id}**
  Establish a WebSocket connection to receive real-time updates while a scan is in progress.

Detailed OpenAPI documentation is automatically generated by FastAPI and can be accessed interactively at `/api/docs` when the backend is running.
